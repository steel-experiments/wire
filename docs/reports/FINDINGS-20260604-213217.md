## Findings

1. Session lifecycle is still broken.

   All provided session files, including the failed GitHub Trending session, are still persisted as `status: "ready"` with no terminal update. Example files:

   - `/Users/nikola/.wire/state/sessions/session_d7eed9a2-f0be-4899-beae-b44a4bbaf515.json`
   - `/Users/nikola/.wire/state/sessions/session_328f02d2-94f1-4d3d-b9e0-17af6e7b676f.json`

   Fix belongs in core: every run exit path must explicitly release or close the browser session unless the task asks to keep it open, and persist the final session state either way.

2. Run persistence is too weak for review and recovery.

   The saved run files omit direct `sessionId`, step count, review failures, and event pointers. The final `result` is also stored as an escaped JSON string instead of structured JSON. Example files:

   - `/Users/nikola/.wire/state/runs/run_82d489db-47b7-4b1e-b8ef-5b2355b5d098.json`
   - `/Users/nikola/.wire/state/runs/run_ef53a606-75d5-4138-8c8f-ee811499e663.json`

   Fix belongs in core: persist structured result payloads plus run/session/event/artifact linkage.

3. The coarse completion contract is too easy to satisfy.

   HN and IMDb both reached "5 items", then artifact review correctly rejected quality. HN counted comment text as search results with empty `date` and `points` fields. IMDb returned five rows but every title is empty.

   Fix belongs in core/helper: item contracts need required field completeness and non-empty checks, not just array length.

4. Artifact-review failures do not drive repair.

   Booking review identified a missing rating and missing free-cancellation evidence, but the run still exhausted steps instead of producing a corrected artifact. The Booking artifact has rank 2 with `rating: ""` and no free-cancellation proof in the URL:

   - `/Users/nikola/.wire/state/artifacts/artifact_75bf534f-d3ab-45f8-9874-f9d51ae3cf87-output.json`

   Fix belongs in core: failed artifact review should become a concrete repair contract for the next action, not just a note.

5. Multi-site extraction lacks a durable scratchpad/checklist.

   Pricing visited Linear and Asana, then Monday, but final persisted output is only an Asana dump. Linear evidence exists separately, but was not integrated into the final table:

   - `/Users/nikola/.wire/state/artifacts/artifact_17387f14-2f61-4df6-92e7-d0cfd7374785-output.json`
   - `/Users/nikola/.wire/state/artifacts/artifact_8041332a-bc40-4d83-8b58-fd51885e6b7b-output.json`

   Fix belongs in helper/core: add a small structured collection helper for "visit N sources, fill schema rows, then render final table."

6. GitHub Trending failed before a run was written.

   There is a session file, but no completed run file and no event records were found tied to that session ID. Early failures can therefore lose the most important audit trail.

   Fix belongs in core: create and persist the run before session/browser startup work, then mark it failed or aborted with the startup error.

## Priority Fixes

1. Core session finalization with explicit "keep open" semantics.
2. Persist structured run results and full run/session/event/artifact links.
3. Upgrade completion contracts from count-based to schema/quality-based.
4. Convert artifact-review failures into mandatory repair instructions.
5. Add a generic structured extraction helper for list/table tasks.
6. Ensure early startup failures always produce a run record.
