# Wire by Steel

The zero-weight browser agent.

The web is not an API.
It is state, latency, auth, pixels, JavaScript, downloads, dialogs, blockers, and consequences.

Agents need to work there.
Not in a simulation of it.

Wire exists to make that possible without building another heavy browser framework.

## Our Belief

The browser should be real.
The agent should be small.
The evidence should be permanent.

Steel carries the browser weight.
Wire carries intent through it.

## Principles

Keep the core small.
Every new core concept compounds forever.

Separate what must not be mixed.
Profiles are not tasks.
Traces are not memory.
Policy is not prompting.
Skills are not diaries.
Helpers are not a framework.

Use code as the action language.
Clicks are too small.
Screenshots are too vague.
Code can be read, patched, replayed, and blamed.

Keep helpers thin.
Abstractions should reduce noise, not hide causality.
Every shortcut needs an escape hatch.

Treat evidence as the product.
A run is not complete because the agent says so.
It is complete when the artifacts prove what happened.

Prefer learning over luck.
A lucky success that teaches nothing is weak.
A failed run that exposes the cause is progress.

Make uncertainty explicit.
When one more run can answer a real question, branch.
Compare paths.
Reject hypotheses.
Find counterexamples.

Store the map, not the diary.
Durable site knowledge belongs in skills.
Secrets, transcripts, and one-off narration do not.

Keep policy outside the loop.
The model may propose.
The system decides what is allowed.

## What We Refuse

We refuse giant agent frameworks.
We refuse screenshot-only automation.
We refuse hidden retries.
We refuse opaque self-healing.
We refuse prompt soup.
We refuse demos that erase failure.
We refuse clever cores.

## The Standard

A user brings a task, workflow, or hunch.

Wire returns one of three things:

- the work completed with evidence,
- the failure diagnosed with evidence,
- the next experiment justified by evidence.

## The Line

Make the browser real.
Make the core small.
Make actions inspectable.
Make failures useful.
Make lessons durable.

That is the zero-weight browser agent.
