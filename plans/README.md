## Findings considered and rejected

So nobody re-audits these:

### From the 2026-06-10 pass

- **sk-ant API key not redacted** — STALE. A security sub-audit tested the old
  regex; `src/shared/redact.ts:8` now allows `-`/`_` and explicitly covers
  `sk-ant-api03-…`. Fixed as alignment item A2 (commit `7f97e6b`).
- **Skill-injection sanitizer is bypassable** — real, but already tracked as
  alignment item **B7** (stance: the policy engine is the trust boundary;
  document the limitation honestly, no new denylist).
- **`reverse().find()` × ~24 sites; `progressLedgerFromEvents` called more than
  once per finish** — real patterns, but micro-optimizations in a network-bound
  agent loop dominated by multi-second LLM/browser round-trips. Poor leverage;
  the readability/dedup angle is already inside alignment item **C3**.
- **`displaySafeUrl` unwired, `cloneJson` / `HELPER_PREAMBLE` dead code, verdict
  module extraction, three identical signal interfaces, agent↔eval import cycle,
  experiments module untested** — all already in the alignment plan (**C4, C3,
  C2, A10/A13**). The agent↔eval allowlist in `scripts/architecture-check.mjs`
  is intentional and slated to be burned to zero (C2).
- **Contract-reprompt does not push a `thought-summary` guidance event**
  (`finish-flow.ts:173`) — the `contract-check` event *is* pushed to the trace
  (lines 166-172) and the step-count increment mirrors the extraction-reprompt
  path (137-149); looks by-design, not a defect.
- **Non-null assertion in `turn.ts`** — currently provably safe; a smell, not a bug.

### From the 2026-07-17 pass

- **Exec-risk regex gating evadable / no domain allowlist**
  (`src/policy/rules.ts:328-361`) — consistent with the documented "policy
  engine is the trust boundary; document, don't denylist" stance (same class
  as B7). Informational; a real boundary would be CDP-level interception — a
  design effort, not a patch.
- **Navigating-click inference can mask a post-click failure**
  (`src/providers/browser/steel/provider.ts:67-69,332-344` — keys on exec-wide
  `wireEvents` instead of per-target evidence) — real but narrow (multi-target
  exec or post-click crash), LOW-MED confidence it ever misfires live, and
  this path was just stabilized after the 2026-06-30 live-test bug; fix risk
  outweighs the demonstrated harm. Revisit if a live run shows a false
  `{navigated:true}` masking an error.
- **`pnpm audit` non-functional** — pnpm 10.33 calls npm's retired audit
  endpoints (HTTP 410). Not actionable in-repo; runtime dep surface is `zod`
  only after plan 004.
- **`lint` script aliases `typecheck`; no linter/formatter** — consistent with
  the zero-dep doctrine; deliberate.
- **METRICS.md updates are honor-system** — currently accurate (last row
  matches the last commit day); automation would add CI weight for a
  discipline that is holding. Revisit if it rots.
- **Console outside `scripts/architecture-check.mjs`** — by-design per
  `console/CLAUDE.md` (separate app, own budgets). The CI gap was what
  mattered (plan 008).
