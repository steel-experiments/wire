## Why this matters

CI on `main` has been red since 2026-06-30. `hls.js` was added to the root
`package.json` without regenerating `pnpm-lock.yaml`, so the CI step
`pnpm install --frozen-lockfile` fails with `ERR_PNPM_OUTDATED_LOCKFILE`
before any check runs (confirmed in the logs of GitHub Actions run
28478860328 and the three runs before it). On top of that, `hls.js` is in the
wrong manifest: its only import in the entire repo is
`console/src/components/recording-player.tsx:5`, and the console is a separate
Bun app with its own `console/package.json` тАФ which does NOT declare it. The
root package is the published zero-dependency agent (`zod` is the only
sanctioned runtime dep per `docs/proposals/PROGRAM.md`). Finally, a stray
`bun.lock` at the repo root disagrees with `pnpm-lock.yaml` (the root
`bun.lock` contains `hls.js`; the pnpm lockfile does not) тАФ the repo's package
manager is pnpm (`"packageManager": "pnpm@10.33.0"`).

After this plan: root manifest is back to `zod` only, CI installs cleanly,
the console declares its own dependency, and there is exactly one lockfile
per package root.

`cd console && bun install`.

**Verify**: `grep -c '"hls.js"' console/bun.lock` тЖТ at least `1`.

**Verify**: `git check-ignore bun.lock` тЖТ prints `bun.lock`;
`git check-ignore console/bun.lock` тЖТ exits non-zero (NOT ignored).
